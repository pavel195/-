Этот файл — краткая карта документации для 16 практических задач из тренажёра. Сначала смотри общий блок, затем раздел по конкретной задаче.

## Алгоритмы и структуры данных

### 1. Ближайшая большая высота

Что читать:
- Python data structures: раздел про списки как стек.
- Сложность операций Python-контейнеров.

Что нужно понять:
- монотонный стек индексов;
- почему каждый элемент кладётся и снимается со стека не более одного раза;
- почему решение работает за `O(n)`.

Минимальный план решения:
1. Идти по массиву справа налево.
2. Удалять из стека все здания не выше текущего.
3. Если стек не пуст, верхушка — ближайшее большее здание справа.
4. Записать расстояние между индексами.

### 2. Самый частотный символ

Что читать:
- Python data structures: словари и множества.
- Built-in functions: `max`, `sum`.

Что нужно понять:
- нормализация регистра через `lower` или `casefold`;
- фильтрация только букв;
- подсчёт частот через `dict`;
- обработка пустой строки.

Минимальный план решения:
1. Пройти по строке.
2. Учитывать только латинские буквы.
3. Считать частоты в словаре.
4. Найти максимум.
5. Сложить частоты всех символов, у которых частота равна максимуму.

### 3. k-й элемент двух отсортированных массивов

Что читать:
- `bisect` как справку по идее бинарного поиска.
- Python data structures: индексация списков.
- TimeComplexity.

Что нужно понять:
- почему нельзя сливать массивы;
- как отбрасывать часть одного массива;
- почему шаг уменьшает `k`;
- граничные случаи: один массив пустой, `k == 1`.

Минимальный план решения:
1. Хранить текущие индексы в обоих массивах.
2. Сравнивать элементы около `k // 2`.
3. Отбрасывать часть массива, которая точно меньше k-го элемента.
4. Повторять до базового случая.

### 4. Слияние и сортировка слиянием

Что читать:
- Python sorting HOWTO.
- Python data structures: списки и срезы.

Что нужно понять:
- слияние двумя указателями;
- рекурсивное деление массива;
- разница между собственным merge sort и встроенной сортировкой;
- память `O(n)` и время `O(n log n)`.

Минимальный план решения:
1. Реализовать `merge_sorted_times`.
2. В `merge_sort_times` делить список пополам.
3. Рекурсивно сортировать половины.
4. Объединять через `merge_sorted_times`.

### 5. Бинарный поиск и метод бисекции

Что читать:
- `bisect`.
- Built-in functions.

Что нужно понять:
- поиск первого вхождения через смещение правой границы;
- инварианты `left`, `right`, `mid`;
- метод деления отрезка пополам для функции с разными знаками на концах;
- точность `eps`.

Минимальный план решения:
1. Для массива хранить `answer` и искать левую границу.
2. Для корня проверять разные знаки на концах.
3. Пока длина отрезка больше `eps`, делить его пополам.
4. Оставлять половину, где знак меняется.

### 6. Проверка вложенности тегов

Что читать:
- Python data structures: список как стек.
- `re`: регулярные выражения.

Что нужно понять:
- стек открывающих тегов;
- различие `<tag>` и `{tag}`;
- закрывающий тег должен совпадать с верхушкой стека;
- пустой стек в конце означает корректную вложенность.

Минимальный план решения:
1. Регулярным выражением найти только теги.
2. Открывающий тег положить в стек.
3. Закрывающий сравнить с верхушкой.
4. При несовпадении вернуть `False`.
5. В конце проверить, что стек пуст.
